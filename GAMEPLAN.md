# 타자성배틀 — 게임 설계 목표

## 핵심 비전
- 한글 타자 기반 실시간 성 대결 게임
- 모든 시각 요소를 **타일맵 에셋**으로 구현 (Canvas 직접 드로잉 최소화)
- 단어마다 고유한 효과 + 숨겨진 이스터 에그로 재미 추가
- 유닛 간 **가위바위보 상성** 관계로 전략성 부여

## 유닛 상성 (가위바위보)
```
검사  ▶ 마법사  (근접으로 마법사 제압)
마법사 ▶ 기사   (마법이 갑옷 관통)
기사  ▶ 궁수   (두꺼운 갑옷으로 화살 버팀)
궁수  ▶ 검사   (원거리에서 먼저 처리)

투석기 ▶ 건물   (건물 특화 고데미지)
기사  ▶ 투석기 (빠른 접근으로 제압)
```
- 상성 유닛에게 **공격력 1.5~2배** 적용
- 상성 피격 시 **피격 데미지 0.6배** 적용

## 이스터 에그 (숨겨진 단어)
입력했을 때 예상 밖의 효과 발동. UI에 힌트 없음.

| 단어   | 효과                                         | 타겟     |
|--------|----------------------------------------------|----------|
| 달걀   | 유닛 한 명에게 초대형 데미지 (계란으로 바위치기) | 적 유닛  |
| 고양이 | 적 유닛 전체 **매혹** — 잠시 아군 쪽 공격 중단 | 전체     |
| 잠     | 적 유닛 전체 **수면** — 일정 시간 움직임 정지  | 전체     |
| 폭풍   | 화면 전체 **범위 데미지**                     | 적 전체  |
| 신     | 아군 유닛 **무적** 3초                        | 아군 전체 |
| *(추가 예정)* | |  |

## 건물 구현 계획

### 성벽 (wall)
- **타일 기반** 렌더링으로 구현
- 적 유닛이 성벽 HP 소진 후 통과
- 타일: tiny-dungeon 또는 tiny-town 에서 좌표 지정 예정

### 망루 (tower)
- 타일 구현 여부 미정
- 범위 내 적 자동 공격 기능은 ✅ 구현됨

### 병영 (barracks)
- 제외

---

## 에셋 구현 목표
- 모든 투사체 → 타일 스프라이트 우선, 없으면 Canvas 폴백
- 이펙트(화염·얼음·번개 등) → `effects.js`에서 `sprite: { sheet, row, col }` 지정
- 성벽 → 타일 그리드로 렌더링
- 유닛 → 현재 타일맵 방식 유지, catapult 스프라이트 추가 필요
